# Support

Stepper is maintained as an open source shadcn-style component.

## Where to Ask

- Use bug reports for reproducible broken behavior.
- Use feature requests for API, UX, or component ideas.
- Use docs issues when examples, installation steps, or explanations are
  unclear.
- Use the security policy for vulnerabilities.

## Response Expectations

There is no guaranteed response time, but clear reports are much easier to
review. Include reproduction links, screenshots, logs, environment details, and
the exact behavior you expected when they are relevant.

For usage questions, check the README and docs site first:

- https://github.com/francozeta/stepper#readme
- https://francozeta-stepper.vercel.app
