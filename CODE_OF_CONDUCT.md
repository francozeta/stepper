# Code of Conduct

Stepper should be a welcoming project for contributors and users.

## Expected Behavior

- Be respectful and constructive.
- Assume good intent, and ask clarifying questions before escalating.
- Keep feedback focused on the work, not the person.
- Be patient with people who are new to the project or ecosystem.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Personal attacks or repeated bad-faith arguments.
- Publishing private information without permission.
- Derailing issues or pull requests in a way that makes collaboration harder.

## Scope

This code of conduct applies to project spaces, including issues, pull
requests, discussions, comments, and other collaboration channels connected to
Stepper.

## Reporting

If you see behavior that makes the project less safe or welcoming, contact the
maintainer through GitHub. If the report involves sensitive details, avoid
posting those details publicly.

The maintainer may remove comments, close issues, block accounts, or take other
reasonable action to keep the project healthy.
